2.2 Variables and Strings

What is a variable?

  • Names to keep track of values
  • Assignment statement (Name = Expression)
  • Example...speed_of_light = 299792458

What does it mean to ASSIGN a VALUE to a variable?

  • The variable name contains the value.
  • Variables can remaine constant, but values can change.
  • Assignment is important because variables must have value or they don't work.

What is the difference between what the equals "=" means in math versus in programming?

  • Equals in variables means "assignment", not a comparison of value.
  • This does not mean equal, it's more like an arrow pointing to the value.
  • Don't start a variable name with a capital letter.

Strings

  • 'I am a string!' OR;
  • "I am a string!"
  • Double quotes allow for use of single quotes within a string.
  • You can combine strings with operators (like +)
  • Example would be...print 'Hello ' + name

Indexing Strings

  • String is a sequence of characters.
  • We can extract characters from the string by using [] square brackets.
  • We could use...'udacity'[0] to return the "u".
  • We can do this with the number of characters present.
  • The first character is indexed at zero.
  • We can back through a string using negative numbers.
  • Remember...the computer always does the math first in a sequence.
  • Example "string" [expression] = one-character string w/ expression being a number.

Selecting Sub-Sequences

  • Example..."string"[expression]:[expression] with both expressions being numbers.
  • String the is a subsequence of the characters in the string.
  • First expression is position start, second is stop minus 1.
  • We can leave a colon empty to search from start to end or vis versa.
  • We can leave both sides blank to get the whole word.

"Find" Function w/in Strings

  • Find is a method (built in feature of Python).
  • It operates on strings.
  • "string".find("string").
  • Find will give us the fist position in the search string where the target string occurs.
  • If nothing is found the output will be -1.
  • Find cares about capitalization so be specific

Find...Parameters

  • "string".find("string"), number.
  • Passes whats in the search string after the number.
