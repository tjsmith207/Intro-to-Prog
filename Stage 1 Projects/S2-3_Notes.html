2.3 Inputs - Function - Outputs

Inputs & Outputs

  • Data collected by the computer to support functions.
  • Data could be output by the function after some action.

Functions Defined

  • Fundamental programming element that allows for efficient programming.
  • Functions do work within the application and are called repetitively.
  • We used ".find" which is a function.
  • "Functions" and "Procedures" are the same thing.

Function Syntax

  • Functions are "defined" using "def" in Python syntax.
  • Function syntax def function_name(input, input, ...).
  • Indenting within functions defines the function as well.
  • Where you un-indent is where the function ends.
  • You can call a function using function_name() syntax.
  • Functions create a local scope so.
  • Variables you make within the function don't exist outside the function.
  • Once functions run, internal variables are destroyed.
  • Example - string.find(input, input).
  • We will not use objects (like string above) on functions.
  • Example - def rest_of_string(s): then return s[1:].
  • Example - print rest_of_string('audacity') from sample code.

Function Parameters

  • Map inputs to outputs (math function definition).
  • Avoid Global variables that exist outside of functions.
  • Inputs can also be called operands or arguments.
  • Parameters are values that can be passed into a function.
  • Example parameter...function_name(first parameter, second parameter).
  • The function will define how many parameters it needs (1st, 2nd).
  • Passing in less than what it needs will result in an error.

Mutable vs. Immutable Objects

  • An Immutable Object is an object whose state cannot be modified after creation.
  • A Mutable Object can be modified after it is created.
  • Examples of Immutable objects are strings, ints, floats, etc.
  • An immutable object is one that doesn't change when pointers change.
  • The example was when a = 5 then a = 6. The pointer moves to 6, but the value 5 is still 5.
  • Lists are examples of mutable objects.
  • The a list array can be the same with values chaning within the array.

Print vs. Return Statements

  • Print simply prints to the console.
  • Print does not alter the program or function.
  • Return ends a function and brings the value to an external variable.
  • Return does not print anything to the console.
  • "None" return values usually means you haven't returned anything.
  • Return statement alway ends the function.

Recursion

  • Useful to do repeat calculations through the same function.
