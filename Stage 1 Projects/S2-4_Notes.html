2.4 Control Flow and Loops - If and While

Making Decisions

  • Comparison operators less than, greater than, less/greater or equal, not equal, equal.
  • Output of a compare is a boolean value 'true' or 'false'.
  • We use a double == for compare because a single = means assignment.

"IF" Statements

  • Syntax...if "TestExpression": followed by a "block".

"Or" in an "If" Statement

  • If the first expression evaluates to True, then the value is true and the second expression is not evaluated.
  • If the first expression evaluates to False, then the value is the value of the second expression.

Alan Turing

  • Lived from 1912 to 1954
  • Developed abstract model of a computer (1936) called "Turing Machine".
  • Proved that a machine with a few simple operations could simulate any other machine.
  • Worked at Blechley Park breaking the Enigma Encryption Code called Turings Bombe used to break Enigma messages.
  • With arithmetic, comparisons, procedures, and if statements you can write every possible computer program.

While Loops

  • Syntax...while "Test Expression": do block.
  • Block executes until test expression is true.
  • Activity bounces from test expression to block until test expression is false.
  • Break Statement w/in While..."IF" statement that tests to stop the loop.
  • The word "break" is that actual syntax term.
